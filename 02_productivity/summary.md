My tool, Census Insight Reporter, will help users explore U.S. Census Economics 2022 data through a Shiny app. The user will choose a geography (like a state or county) and select one or more economic variables. The app will send a request to the Census API, retrieve the data, and display it in a table (and possibly a simple plot) so the user can see the numbers clearly.

After the data is displayed, the app will generate a short “reporter-style” summary using an AI model (OpenAI or Ollama). The report will explain what the chosen variable means, highlight the most important numeric results, and provide a plain-language takeaway. This reduces the time the user spends translating raw numbers into readable insights.

This design meets the needs of both stakeholders. For the instructor/TA, the tool clearly demonstrates API integration, Shiny functionality, and AI reporting in one workflow. For the student user, the tool makes it easier to interact with Census data and quickly produce a readable summary without manually writing it from scratch. One challenge I expect is handling missing values or API errors cleanly, so I plan to include error handling that shows useful messages when the API request fails or returns incomplete data.
